Lootwhore Config XML Layout

 

<lwconfig>

 <!--You can skip any of the subnodes in settings, or even skip settings entirely.  If a setting is not specified, the values here will be used as default.-->

            <settings>

 

<!-- List of container indices lootwhore can use for autostore. -->

<storebags>5,6,7,8,10,11,12</storebags>

 

<!--Must be enabled or disabled.  Determines if lootwhore will automatically combine stacks in inventory. -->

<autostack>enabled</autostack>

 

<!-- This is primarily used for darkstar or topaz servers.  List bag indices seperated by comma.  Lootwhore will treat any listed bags as if you have access to them instead of honoring the memory checks. -->

                        <forceenablebags></forceenablebags>

 

 <!-- When enabled, lootwhore can store in storage while at a nomad moogle.  This still requires storage to be in the storebags list.  Note that you cannot legitimately access storage when at a nomad moogle. -->

                        <nomadstorage>disabled</nomadstorage>

 

                        <!-- Determines the maximum amount of times lootwhore will try to pass or lot an item if the server fails to tell you the pass or lot succeeded.  Note that the server not telling you does not mean it did not succeed, so an excessively high value is likely not helpful. -->

                        <maxretry>3</maxretry>

 

            </settings>

 

            <!-- A whitelist is used to determine which characters smartpass will trigger on when in listonly mode.  Simply list each character name in an entry tag.  You can skip this if you do not plan to use listonly mode. -->

            <whitelist>

                        <entry>Atomos</entry>

                        <entry>Thorny</entry>

            </whitelist>

 

</lwconfig>
